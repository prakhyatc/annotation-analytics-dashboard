An end-to-end analytics project built on top of an NLP annotation platform developed at the IDIR Lab (UT Arlington). The dashboard demonstrates how operational KPIs, workflow bottlenecks, and annotator performance metrics can be surfaced from annotation-platform data using SQL, Python, and Tableau.

---

# Problem Statement

Managing multiple annotators across concurrent NLP research projects creates operational blind spots:

* No visibility into daily and weekly throughput versus project targets
* No early warning for quality degradation under high annotation volume
* No comparison of task complexity across annotation workflows
* No standardized performance metrics across annotator experience levels

This project addresses these challenges through structured SQL analysis, automated KPI generation, and interactive dashboards.

# Key Business Insights

| Insight                    | Finding                                                                                        |
| -------------------------- | ---------------------------------------------------------------------------------------------- |
| 📈 Throughput Ramp-Up      | All 3 projects scaled from approximately 500 to 3,500–4,000 annotations per week over 20 weeks |
| ⚠️ Bottleneck Task         | Claim verification required 69.5s average review time versus 44.5s for relevance labeling      |
| 🎯 Top Performer           | Highest efficiency score maintained while preserving strong annotation quality                 |
| 🔴 Fatigue Signal          | 16 high-risk days detected where high annotation volume coincided with lower quality           |
| 📉 Experience Gap          | Junior annotators averaged lower accuracy than senior annotators                               |
| 🏁 Forecasting Opportunity | Project-level throughput trends can be used to estimate completion timelines                   |

# How to Run

```bash
# Clone repository
git clone https://github.com/prakhyatc/annotation-analytics-dashboard

# Enter project directory
cd annotation-analytics-dashboard

# Create virtual environment
python3 -m venv venv

# Activate environment
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Generate synthetic dataset
python data/generate_data.py

# Run analysis and export dashboard files
python analysis/run_analysis.py
```

Load the exported CSV files from:

```text
dashboard_exports/
```

into Tableau Public to recreate the dashboard.
