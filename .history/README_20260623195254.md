# Annotation Analytics Dashboard
## Dashboard Preview

![Dashboard](assets/dashboard_screenshot.png)
> **SQL · Python · Tableau/Power BI · KPI Tracking · Data Visualization**

An end-to-end analytics project built on top of the NLP annotation platform I developed at the **IDIR Lab (UT Arlington)**. The dashboard demonstrates how operational KPIs, workflow bottlenecks, and annotator performance metrics can be surfaced from annotation-platform data.

---

## Problem Statement

Managing 8 annotators across 3 simultaneous NLP research projects creates operational blind spots:
- **No visibility** into daily/weekly throughput vs. project targets
- **No early warning** for quality degradation under high volume (annotator fatigue)
- **No comparison** of task complexity across different annotation types
- **No accountability metrics** for annotator performance across experience levels

This dashboard answers those questions using structured SQL queries and interactive visual reports.

---

## Key Business Insights

| Insight | Finding |
|---|---|
| 📈 **Throughput ramp-up** | All 3 projects scaled from ~500 to 3,500–4,000 annotations/week over 20 weeks |
| ⚠️ **Bottleneck task** | `claim_verification` takes **69.5s avg** vs 44.5s for `relevance_labeling` — 56% slower |
| 🎯 **Top performer** | James Liu (Senior) — efficiency score 73.9, 94%+ accuracy sustained across 20 weeks |
| 🔴 **Fatigue signal** | 16 HIGH RISK days detected where high volume (>150 annotations) correlated with accuracy <82% |
| 📉 **Junior gap** | Junior annotators average 81% accuracy vs 94% for seniors — 13pp gap suggesting targeted training need |
| 🏁 **Project forecast** | At current pace, ClaimScore Survey on track to complete 3 weeks ahead of schedule |

## How to Run

```bash
# 1. Clone and install dependencies
git clone https://github.com/prakhyatc/annotation-analytics-dashboard
cd annotation-analytics-dashboard
pip install pandas numpy matplotlib

# 2. Generate synthetic dataset
python data/generate_data.py

# 3. Run analysis and generate charts
python analysis/run_analysis.py

# 4. (Optional) Load into SQLite for raw SQL queries
sqlite3 data/annotations.db
.mode csv
.import data/annotations.csv annotations
.import data/projects.csv projects
# Then run queries from sql/analysis_queries.sql

# 5. Load dashboard_exports/*.csv into Tableau Public or Power BI Desktop
```

## Background

This project is an analytics layer built on top of the [IDIR Lab NLP annotation platform](https://github.com/prakhyatc) I developed at UT Arlington (Jan 2025 – Present). The platform consolidates 15+ legacy PHP scripts into a unified SQL-backed system supporting 50K+ structured annotations across 3 research projects. This dashboard transforms the raw operational data into stakeholder-facing KPIs and actionable workflow insights.

---

## Interactive Dashboard

The dashboard visualizes annotation productivity and quality across three NLP annotation projects. It includes weekly throughput trends, annotator performance, task complexity, accuracy patterns, and bottleneck alerts.

Key metrics:
- 168,434 total annotations
- 8 active annotators
- 3 annotation projects
- 87.4% average accuracy
- 50 alert days detected, including 16 high-risk days

Dashboard views include:
